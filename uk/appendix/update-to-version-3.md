# Оновлення на версію 3.х

У першу чергу це оновлення торкнулося старих версій браузерів, точніше, тепер підтримуються наступні версії:

* Internet Explorer 9+
* Chrome, Edge, Firefox та Safari — завдяки вбудованій системі оновлень у ці браузери підтримується поточна та попередня версії браузерів
* Opera — у фаворитах не ходить, тому підтримують лише поточну версію
* Safari Mobile iOS 7+
* Android 4+

Крім цього, було внесено багато змін, які ламають зворотну сумісність із попередніми версіями. Усі їх перелічувати не буду, зупинюся на тих, з якими вже довелося зіткнутись.

> Більшість наведених змін стосується не тільки версії 3.x, але й старих гілок, починаючи з версій 1.12.x та 2.2.x

## AJAX <a href="#ajax" id="ajax"></a>

*   Метод `jQuery.ajax()` тепер сумісний з Promise і ви можете використовувати методи `then()` та `catch()`:

    <pre class="language-javascript"><code class="lang-javascript">$.ajax({url:"/get-my-page.html" /* і т.д. */ })
    <strong> .then(function() { /* все ОК */ })
    </strong> .catch(function() { /* помилка */ })
    </code></pre>
* Додано нову сигнатуру для виклику двох AJAX-методів `$.get(settings)` та `$.post(settings)`, тепер налаштування сумісні з `$.ajax(settings)`.
* При підключенні скриптів з іншого домену тепер в обов'язковому порядку вимагає вказівки `dataType: "script"`
* При виконанні AJAX-запитів і вказівці URL з хешем останній більше не обрізається, а відправляється на сервер як є.

## Атрибути <a href="#attributes" id="attributes"></a>

*   Раніше метод `removeAttr()` для `true`/`false` атрибутів типу «`checked`», «`selected`» та «`readonly`» потихеньку виставляв відповідну властивість DOM-елемента в «`false`», тепер, будь ласка, робити це ручками:

    ```javascript
    $("input[type=email]").removeAttr("readonly").prop("readonly", false)
    ```
* Якщо викличете метод `val()` для мультиселекта, в якому нічого не вибрано, то отримаєте у відповідь порожній масив, а раніше був `null`
* Для SVG запрацювали методи по маніпуляції з класами (хоча повноцінної підтримки SVG в jQuery немає).

## Core <a href="#core" id="core"></a>

* Ядро jQuery тепер запускається в [strict mode](https://uk.javascript.info/strict-mode).
* Обробники події `document-ready` тепер запускаються асинхронно, тобто якщо якийсь обробник зафейлив, то це не вплине на запуск інших обробників.
* Метод `$.isNumeric()` більше не намагається кастувати метод `toString()` для довільних об'єктів (кому це було треба?).
* Методи `width()`, `height()` і так далі; раніше, викликавши їх для порожньої колекції об'єктів, ми отримували `null`, тепер – `undefined`.
*   Офіційно додано promise `jQuery.ready`, який дуже зручно заюзати разом з `$.when()`:

    ```javascript
    $.when($.ready, $.getScript("script.js") )
     .then(function() {
       // документ готовий, і скрипт script.js підвантажений
     })
     .catch( function() {
       // помилка
     });
    ```
* Метод `jQuery.unique()` перейменовано на `jQuery.uniqueSort()`
* Метод `jQuery.parseJSON()` застарів, переходьте на `JSON.parse()`
* Унутрішнє сховище даних перейшло на camelCase, це важливо для тих, хто використовував його, не вдаючись до методу `data()`

## Об'єкт Deferred <a href="#deferred" id="deferred"></a>

Як я вже згадував раніше, об'єкт Deferred тепер сумісний з Promise з ES-2015, а це нам загрожує переписуванням методів `done()` на `then()` та `fail()` на `catch()`.

Другий важливий момент — callback-функції, згідно зі специфікацією ES-2015, мають приймати лише один аргумент: для успішного виконання це деякий результат виконання, у разі виникнення помилки функція це буде сама помилка. Якщо у вас не виходить так зробити, то старі функції `done()` та `fail()` все ще залишаються з нами, хоч чую, і їх скоро випиляють:

```javascript
// було
$.get("/get-my-page.html")
 .done(function(data, textStatus, jqXHR) { /* все ОК */ })
 .fail(function(jqXHR, textStatus, errorThrown) { /* помилка */ });

// стало
$.get("/get-my-page.html")
 .then(function(data) { /* все ОК */ })
 .catch(function(error) { /* помилка */ });
```

## Розміри <a href="#size" id="size"></a>

Невеликі зміни спіткали функції `width()`, `height()`, `css("width")`, та `css("height")`, тепер вони можуть повертати не тільки integer значення висоти та ширини, але й float, така точність пов'язана з переходом на використання `getBoundingClientRect()`.

Ще момент, виклики `$(window).outerWidth()` та `$(window).outerHeight()` тепер містимуть розміри скролбарів вікна.

## Ефекти <a href="#effects" id="effects"></a>

Анімація переїхала на використання requestAnimationFrame API, тож тепер усе стало швидше та гарніше.

Функції `.show()`, `.hide()` та `.toggle()` навчилися запам'ятовувати попередній стан CSS-властивості `display`.

Кількість аргументів easing-функцій скоротили до одного аргументу, який містить прогрес анімації від 0 до 1.

## Події <a href="#events" id="events"></a>

Видалено shorthand-методи для наступних подій: `load()`, `unload()` та `error()`; пов'язана ця зміна з конфліктами, що виникають при використанні цих методів, тож переписуйте на `on()`:

```javascript
// було
$("img").load(fn);

// стало
$("img").on("load", fn);
```

Видалено синтетичну подію `ready`, тож `on("ready", fn)` більше не працює, використовуйте синтаксис `$(fn)`.

Делеговані події, у випадку, якщо їх намагаються навісити з використанням неправильних селекторів, тепер одразу лаятимуться і викидатимуть помилку. Дебажити стане легше:

```javascript
// приклад зламаного селектора div:not
$("body").on("click", "div:not", e => false);
```

## Селектори <a href="#selectors" id="selectors"></a>

За селектори `:hidden` та `:visible` тепер відповідає `getClientRects()`; якщо в запитуваного елемента є layout box, значить, він вважається видимим. Як результат, порожній `<span>` або `<br/>` тепер вважаються видимими.

Криві селектори `$("#")` та `find("#")` тепер викликатимуть помилку.

Описав багато, але не все, повне керівництво доступне на офіційному сайті:

* [jQuery Core 3.0 Upgrade Guide](https://jquery.com/upgrade-guide/3.0/)
* [jQuery Core 3.5 Upgrade Guide](https://jquery.com/upgrade-guide/3.5/)
