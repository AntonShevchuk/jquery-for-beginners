# Пишемо свій widget

Відправною точкою при написанні віджета для jQuery UI для вас буде [офіційна документація](https://jqueryui.pbworks.com/w/page/12138135/Widget%20factory), але оскільки зі знанням англійської не в усіх склалося, то я постараюся перекласти й адаптувати інформацію, викладену в ній.

Перше, про що варто розповісти, це те, що правила написання плагінів для jQuery занадто вільні, що не сприяє їхній якості. При створенні jQuery UI, видно, вирішили піти шляхом стандартизації процесу написання плагінів та віджетів. Я не можу сказати, наскільки задум удався, але стало явно краще, ніж було. Почну з опису каркаса для вашого віджета:

```javascript
$.widget("book.expose", {
  // налаштування за замовчуванням
  options: {
    color: "red"
  },
  // ініціалізація widget
  // вносимо зміни в DOM і навішуємо обробники
  _create: function() {
    this.element;    // шуканий об'єкт у jQuery-обгортці
    this.name;       // ім'я - expose
    this.namespace;  // простір – book
    this.element.on("click." + this.eventNamespace, function(){
      console.log("click");
    });
  },
  // метод відповідає за застосування налаштувань
  _setOption: function( key, value ) {
      // застосовуємо зміни налаштувань
      this._super( key, value );
  },
  // метод _destroy має бути антиподом до _create
  // він має прибрати всі зміни, внесені в DOM, і прибрати всі обробники, якщо такі були
  _destroy: function() {
    this.element.off('.'+this.eventNamespace);
  }
});
```

{% hint style="info" %}
Спостережливе око помітить, що всі перелічені методи починаються зі знака підкреслення – це такий спосіб виділити «приватні» методи, які недоступні для запуску. Якщо ми спробуємо запустити `$('#my').expose('_destroy')`, то отримаємо помилку. Але врахуйте – це лише домовленість, дотримуйтеся її!

Для обходу домовленості про приватність можна використовувати метод `data()`:

```javascript
$("#my").data("expose")._destroy() // місце для смайла «(evil)»
```
{% endhint %}

У цьому прикладі я постарався задати гарний тон написання віджетів – я «навісив» обробники подій у `namespace`. Це дасть надалі можливість контролювати те, що відбувається, без необхідності лазити в код віджета. «True story».

> Код, описаний у методі `_destroy()`, надлишковий, оскільки він і так виконується в публічному `destroy()`. Наведений тут для наочності.

А для ледачих, щоб не прописувати кожного разу `eventNamespace` в обробниках подій, розробники додали у версії 1.9.0 два методи: `_on()` та `_off()`. Перший приймає два параметри:

* DOM-елемент, або селектор, або jQuery-об'єкт
* набір обробників подій у вигляді об'єкта

Усі перелічені події будуть «висіти» в просторі `eventNamespace`, тобто результат буде приблизно однаковим:

```javascript
this._on(this.element, {
  mouseover:function(event) {
    console.log("Hello mouse");
  },
  mouseout:function(event) {
    console.log("Bye mouse");
  }
});
```

Другий метод, `_off()`, дозволяє вибірково вимикати обробники:

```javascript
this._off(this.element, "mouseout click");
```

Ну, каркас каркасом, пора переходити до функціоналу. Додамо довільну функцію з довільним функціоналом:

```javascript
callMe:function() {
  console.log("Allo?");
}
```

До цієї функції ми легко зможемо звертатися як з інших методів віджета, так і ззовні:

```javascript
// зсередини
this.callMe()
// ззовні
$("#my").expose("callMe")
```

Якщо ваша функція приймає параметри, то передача аргументів здійснюється наступним способом:

```javascript
$("#my").expose("callMe", "Hello!")
```

Якщо ви хочете достукатися в обробнику подій до методу віджета, то не забудьте про область видимості змінних і зробіть наступний маневр:

```javascript
{
  _create: function() {
    var self = this; // ось він!
    this.element.on("click."+this.eventNamespace, function() {
      // тут використовуємо self, оскільки this вже вказує на
      // елемент по якому клікаємо
      self.callMe();
    })
  }
}
```

Добре йдемо, тепер поговоримо про події. Для більш гнучкої розробки та впровадження віджетів передбачено функціонал зі створення довільних подій та їх «прослуховування»:

```javascript
// ініціюємо подію
this._trigger("incomingCall");

// підписуємося на подію при ініціалізації віджета
$("#my").expose({
  incomingCall: function(ev) {
    console.log("din-don");
  }
})

// або після, використовуючи як ім'я події
// ім'я віджета + ім'я події
$("#my").on("exposeincomingCall", function() {
  console.log("tru-lya-lya")
});
```

Матеріалу багато, я розумію, але ще додам опис кількох методів, які можна викликати із самого віджета:

`_delay()` – ця функція працює як `setTimeout()`, от тільки контекст переданої функції вказуватиме на сам віджет (це щоб не заморочуватися з областю видимості)

`_hoverable()` та `_focusable()` – цим методам необхідно згодовувати елементи, для яких необхідно відстежувати події `hover` та `focus`, щоб автоматично додати до них класи `ui-state-hover` та `ui-state-focus` при настанні цих подій

`_hide()` та `_show()` – ці два методи з'явились у версії 1.9.0, вони створені, щоб стандартизувати поведінку віджетів при використанні методів анімації; налаштування заведено ховати в опціях під ключами «hide» та «show» відповідно. Використовувати методи слід наступним чином:

```javascript
{
  options: {
    hide: {
      effect: "slide",     // налаштування еквівалентні виклику
      duration: 500        // .hide("slide", 500)
    }
  }
}
// всередині віджета слід використовувати виклики _hide() та _show()
this._hide( this.element, this.options.hide, function() {
  // це наша функція зворотного виклику
  console.log('сховали');
});
```

Існує ще пара методів, які реалізовані до нас:

```javascript
{
  enable: function() {
    return this._setOption( "disabled", false );
  },
  disable: function() {
    return this._setOption( "disabled", true );
  }
}
```

Фактично, ці функції створюють синонім для виклику:

```javascript
$("#my").expose({ "disabled": true }) // або false
```

Наше завдання зводиться лише до відстеження цього прапорця в методі `_setOption()`.
