# jQuery UI

jQuery UI являє собою набір віджетів та плагінів від самих розробників jQuery. На мою думку, цей інструмент необхідно вивчити настільки, наскільки це потрібно, щоб не писати свої «велосипеди». Завантажити-почитати про цю надбудову над jQuery можна на домашній сторінці проєкту – [https://jqueryui.com/](https://jqueryui.com/).

Що нам необхідно знати про віджети та плагіни? Перше – це які вони є, і друге – як працюють. На цих двох моментах я й постараюся зупинитися.

## Інтерактивність

Почну з корисних плагінів, які можуть спростити життя при створенні інтерактивних інтерфейсів:

* [Draggable](https://jqueryui.com/draggable/) – цей компонент дозволяє зробити будь-який DOM-елемент перетягуваним за допомогою миші
* [Droppable](https://jqueryui.com/droppable/) – це логічне продовження компонента Draggable; необхідний для роботи з контейнерами, всередину яких можна перетягувати елементи
* [Resizable](https://jqueryui.com/resizable/) – як випливає з назви, дає можливість розтягувати будь-які DOM-елементи
* [Selectable](https://jqueryui.com/selectable/) – дозволяє організувати «вибір» елементів; зручно використовувати для організації менеджменту картинок
* [Sortable](https://jqueryui.com/sortable/) – сортування DOM-елементів

## Віджети

Віджети – це вже комплексне рішення, що містить не тільки код JavaScript, але й певну реалізацію HTML та CSS:

*   Accordion – цей віджет слід використовувати, якщо у тебе в проєкті вже використовується jQuery UI; сам по собі основний його функціонал можна реалізувати в кілька рядків (подивитися можна в [accordion.html](https://anton.shevchuk.name/book/code/accordion.html))

    ![jQuery UI Accordion](../../../.gitbook/assets/accordion.png)
* Autocomplete – як і випливає з назви, цей віджет відповідає за додавання функції автодоповнення до полів введення, звісно, з підтримкою AJAX
*   Button – створення кнопок за допомогою JavaScript – ще той моветон, але, можливо, знадобиться, якщо ти сильно зав'язався на jQuery UI:

    ![jQuery UI Buttons](../../../.gitbook/assets/buttons.png)
*   Datepicker – якщо твій браузер не підтримує повною мірою специфікацію HTML 5 і `<input type="date"/>` зокрема, то знадобиться емуляція цієї можливості за допомогою цього віджета: &#x20;

    ![jQuery UI Datepicker](../../../.gitbook/assets/datepicker.png)
*   Dialog – віджет, призначений для створення злегка незграбних діалогових вікон:

    ![jQuery UI Dialog](../../../.gitbook/assets/dialog.png)
* Menu – створення меню зі списку, з підтримкою вкладеності
*   Progressbar – назва говорить сама за себе, але в HTML 5 він теж включений:

    ![jQuery UI ProgressBar](../../../.gitbook/assets/progressbar.png)
*   Slider – ще один віджет для застарілих браузерів:

    ![jQuery UI Slider](../../../.gitbook/assets/slider.png)
*   Spinner – ще один зручний контрол для форм, знову ж таки, в HTML 5 вже є:

    ![jQuery UI Spinner](../../../.gitbook/assets/spinner.png)
*   Tabs – вони ж таби, досить популярний елемент у web-розробці, і так само, як і «Accordion», цілком замінюваний простим кодом (див. [tabs.html](https://anton.shevchuk.name/book/code/tabs.html)):

    ![jQuery UI Tabs](../../../.gitbook/assets/tabs.png)
* Tooltip – ось і останній віджет, спливаючі підказки; простий і має бути затребуваний, ну а там життя покаже

На цьому огляд віджетів можна вважати завершеним, повернемося до плагінів.

{% hint style="info" %}
Усі віджети та плагіни зав'язані на ядро jQuery UI, але є також залежності між самими плагінами і варто про них пам'ятати. Але не хвилюйся – при збірці jQuery UI пакета всі залежності перевіряються автоматично, тобто коли тобі знадобиться досі не підключений віджет, краще завантажити збірку заново.
{% endhint %}

## Утиліти

Утиліт у нас небагато – ось корисний плагін Position, який дозволяє контролювати положення DOM-елементів – [https://jqueryui.com/position/](https://jqueryui.com/position/), а ще є фабрика зі створення віджетів, але про неї я розкажу трохи пізніше.

## Ефекти

Серед ефектів, що надаються jQuery UI, я виділяю чотири пункти:

* Анімація кольору
* Анімація зміни класів
* Набір ефектів
* Розширення можливостей easing

За анімацію кольору відповідає компонент «Effects Core», який дозволяє анімувати зміни кольору за допомогою методу `animate()`:

```javascript
$("#my").animate({ backgroundColor: "black" }, 1000);
```

Так-так, базовий jQuery не вміє цього робити, а ось jQuery UI дозволяє анімувати наступні параметри:

* `backgroundColor`
* `borderBottomColor`
* `borderLeftColor`
* `borderRightColor`
* `borderTopColor`
* `color`
* `outlineColor`

Ще однією можливістю, що міститься в «Effects Core», є анімація змін класу DOM-елемента, тобто коли ти будеш присвоювати новий клас елементу, то замість звичайного моментального застосування нових CSS властивостей ти спостерігатимеш анімацію цих властивостей від поточних до заданих у присвоюваному класі. Для використання цього функціоналу нам знадобляться старі знайомі – методи `addClass()`, `toggleClass()` та `removeClass()`, з однією лише різницею – при виклику методу другим параметром має бути вказана швидкість анімації:

```javascript
$("#my").addClass("active", 1000);

$("#my").toggleClass("active", 1000);

$("#my").removeClass("active", 1000);
```

Якщо з попереднього абзацу в тебе не виникло розуміння суті того, що відбувається, то цей код для тебе:

```markup
<style>
#my {
    font-size:14px;
}
#my.active {
    font-size:20px;
}
</style>
<script>
$(function () {
    $("#my").addClass("active", 1000);
    // тут виходить аналогічно наступному виклику
    $("#my").animate({"font-size":"20px"}, 1000);
});
</script>
```

> А ще з'являється метод `switchClass()`, який замінює один клас іншим, але мені він жодного разу не знадобився.

Про набір ефектів я не буду довго розповідати, їх краще подивитися в дії на сторінці [https://jqueryui.com/effect/](https://jqueryui.com/effect/). Для роботи з ефектами з'являється метод `effect()`, але сам по собі його краще не використовувати, адже UI розширив функціонал вбудованих методів `show()`, `hide()` та `toggle()`. Тепер, передавши як параметр швидкості анімації назву ефекту, ти отримаєш необхідний результат:

```javascript
$("#my").hide("puff");

$("#my").show("transfer");

$("#my").toggle("explode");
```

> Наведу список ефектів, може, хтось запам'ятає: `blind`, `bounce`, `clip`, `drop`, `explode`, `fold`, `highlight`, `puff`, `pulsate`, `scale`, `shake`, `size`, `slide`, `transfer`.

Пам'ятаєш, у главі про [анімацію](../../40\_animation/) я розповідав про easing та однойменний плагін для jQuery? Так ось, UI теж розширює easing, тож підключивши UI, можна відключати плагін easing. І так, цей функціонал зав'язаний лише на «Effects Core».

## Теми

Однією з найчудовіших особливостей jQuery UI є можливість змінювати «шкурки» всіх віджетів разом, і для цього навіть передбачена спеціальна утиліта – [ThemeRoller](https://jqueryui.com/themeroller/):

![jQuery UI Theme Roller](../../../.gitbook/assets/theme-roller.png)

{% hint style="info" %}
Якщо в якийсь момент часу знадобиться внести зміни в тему, відкрий файл «jquery-ui-#.#.##-custom.css» і знайди рядок, що починається з тексту «To view and modify this theme, visit http:...». Перейди за вказаним посиланням і, використовуючи ThemeRoller, внеси необхідні зміни.
{% endhint %}
