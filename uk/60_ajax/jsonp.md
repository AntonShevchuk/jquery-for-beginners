# Сила JSONP

JSONP — це наш старий знайомий JSON з прошарком у вигляді callback-функції О_о. \
Та годі, давайте на прикладах. Ось як у нас виглядає відповідь сервера у форматі JSON:

```javascript
{
  "note": {
    "time":"2012.09.21 13:12:42",
    "text":"Розповісти, навіщо потрібен JSONP"
  }
}
```

Добре, коли у нас ці дані приходять з нашого сервера — обробимо, і все буде чікі-пікі. Але якщо нам потрібно заполучити дані з іншого сервера, то політика безпеки в браузерах не дозволить відправити XMLHTTPRequest на інший сервер, і треба вже буде щось вигадувати. Можна трішки напрягтися й згадати, що підключати JavaScript з іншого сервера-то ми можемо, і він буде виконаний. Ось вона, зачіпка, а якщо підключуваний скрипт буде містити виклик нашої функції з підготовленими даними — то це вже щось:

```javascript
alertMe({
  "note":{
    "time":"2012.09.21 13:13:13",
    "text":"Який же профіт від використання JSONP?"
  }
})
```

Таким чином, описавши у своєму коді функцію `alertMe()`, ми зможемо обробити дані з віддаленого сервера. Зазвичай сервера ловлять параметр «`callback`» або «`jsonp`» і використовують його як ім'я функції-обгортки:

```markup
<script type="text/javascript" src="https://domain.com/getUsers/?callback=alertMe"></script>
```

Ну, це була передісторія, тепер повернемося до jQuery та методу `ajax()`:

```javascript
$.ajax({
  url: "http://domain.com/getUsers/?callback=?", // вказуємо URL
  dataType: "jsonp",
  success: function (data) {
    // обробляємо дані
  }
});
```

В запитуваному URL спостережливий читач помітить незакінчену структуру «`callback=?`». Отож, замість «`?`» буде підставлено ім'я нової згенерованої функції, всередині якої буде здійснюватися виклик функції `success()`. Замість цієї проксі-функції можна використовувати й свою функцію, достатньо вказати її ім'я як параметр «`jsonpCallback`» при виклику `$.ajax()`.

> А ще варто згадати, що можна вказати, як називається callback-параметр використовуючи параметр `jsonp`. Таким чином, вказавши `jsonp:"my"` в URL, можна буде додати структуру `my=?`.

На даний момент достатньо багато сервісів надають API з підтримкою JSONP:

* [Flickr](https://www.flickr.com/services/api/response.json.html) — робота з пошуком даного сервісу
* [MediaWiki](https://en.wikipedia.org/w/api.php) — відповідно і похідні — Wikipedia, Wiktionary
* [Tumblr](https://www.tumblr.com/docs/en/api/v2) — отримання постів з сервісу

Для ілюстрації роботи спробуємо на зуб API від Flickr:

```javascript
$.getJSON(
    "https://api.flickr.com/services/feeds/photos_public.gne?jsoncallback=?",
    {
        tags: "orange",
        tagmode: "any",
        format: "json"
    },
    function(data) {
        $.each(data.items, function(i, item){
            $("<img/>")
                .attr("src", item.media.m)
                .appendTo("article");
        });
    }
);
```

Використання подібного підходу дозволяє обходити обмеження, що накладаються сервісами на кількість запитів з одного IP, плюс не навантажує сервер додатковою роботою по проксуванню запитів від користувача до сервісів. Саме завдяки цьому принципу мені вдалося створити сервіс [Analyser](https://analyser.hohli.com/).

> На жаль, багато провайдерів сервісів (такі як Google) відмовляють в наданні доступу до їх API з використанням JSONP.
