# Лікуємо JavaScript-залежність

Любов до AJAX буває надмірною і в гонитві за Web2.0 (3.0, 4.0, ... — бажане підкреслити) ми створюємо сайти, в яких всі наші дії біжать через [XMLHTTPRequest](https://uk.wikipedia.org/wiki/XMLHttpRequest). Ні, це, звісно, непогано — знижуємо навантаження на сервер, канал і т.д. і т.п., але є одне «але» — у нас є пошукові машини, які не завжди заморочуються виконанням JavaScript-коду, а контент, захований за AJAX-запитом, їм віддати все-таки потрібно. Відповідно, у нас виникає необхідність дублювання навігації (це як мінімум) для клієнтів без JavaScript.

> Варто пам'ятати, що є ще користувачі, у яких JavaScript вимкнений в браузері (або навіть не підтримується; привіт вам, «Lynx»), але ці знають, що роблять. А ще є скрипти, які ламаються і не дають звичайним користувачам скористатися навігацією по сайту, що користувачів дуже сильно засмучує, тож ця глава не «просто так».

Як же все це обійти і на граблі не наступити? Та все дуже просто — створюйте звичайну навігацію, яку ви б робили, не чувши жодного разу про AJAX і компанію:

```markup
<ul class="navigation">
    <li><a href="/">Home</a></li>
    <li><a href="/about.html">About Us</a></li>
    <li><a href="/contact.html">Contact Us</a></li>
</ul>
<section id="content"><!-- Content --></section>
```

Даний приклад працює у нас без JavaScript, всі сторінки в нашому меню використовують один і той самий шаблон для виведення інформації, і фактично у нас змінюється лише вміст елемента з `id="content"`. Тепер перейдемо до завантаження контенту через AJAX — для цього додамо наступний код:

```javascript
$(function() {
    // навішуємо обробник на всі посилання в нашому меню navigation
    $("ul.navigation a").click(function(){
        var url = $(this).attr("href");   // візьмемо посилання
        url += "?ajax=true";              // додамо до нього параметр ajax=true
        $("#content").load(url);          // завантажуємо оновлений вміст
        return false; // повертаємо false, щоб не спрацював перехід за посиланням
    });
});
```

В даному прикладі ми передбачаємо, що сервер, побачивши параметр `ajax=true` поверне нам не повністю всю сторінку, а лише оновлення для шуканого елемента з `id="content"`.

> Звісно, сервер має бути розумнішим і не вимагати явного вказання для використання AJAX, а повинен цілком задовольнитися, зловивши header «`X_REQUESTED_WITH`» зі значенням «`XMLHttpRequest`». Більшість сучасних фреймворків для web-розробки з цим справляються «з коробки».

Якщо ж керувати поведінкою сервера проблематично і він уперто відправляє нам всю сторінку цілком, то можна написати наступний код:

```javascript
$(function() {
    // навішуємо обробник на всі посилання в нашому меню navigation
    $("ul.navigation a").click(function(){
        var url = $(this).attr("href"); // візьмемо посилання
        // завантажуємо сторінку цілком, але в наш контейнер вставляємо лише вміст #content завантажуваної сторінки
        $("#content").load(url + " #content > *");
        return false; // повертаємо false
    });
});
```
