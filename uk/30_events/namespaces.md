# Простір імен

Як ви вже дізналися, коли ми хочемо створити свій обробник подій, ми пишемо ось такий код:

```javascript
// створюємо свій обробник
$("p").on("click", function() {
    // щось робимо
    alert("Click!");
});
```

Коли нам треба видалити обробники, використовуємо наступний код:

```javascript
// видаляємо всі обробники
$("p").off();

// видаляємо всі обробники click
$("p").off("click");
```

Але, як завжди, є ситуації, коли нам необхідно задіяти або вимкнути не всі обробники (як приклад, треба вимкнути обробку якогось контрола певним плагіном). У цьому випадку нам на допомогу приходять простори імен, і використовувати їх досить легко.

При створенні обробника події додаємо `namespace` через крапку:

```javascript
// створюємо обробник
$("p").on("click.namespace", function(event){
    // щось робимо
    alert("Click: " + event.namespace + "!");
});
```

Коли нам треба викликати обробник події прив'язаний тільки до нашого `namespace` використовуємо наступний синтаксис:

```javascript
$("p").trigger("click.namespace");
```

Коли викликаємо подію з іншого простору імен, наш обробник не буде викликаний:

```javascript
$("p").trigger("click.other");
```

Коли викликаємо всі-всі обробники подій:

```javascript
// викликаємо всі обробники події click
$("p").trigger("click");
```

Коли викликаємо всі обробники без простору імен:

```javascript
// викликаємо всі обробники без простору імен
$("p").trigger("click.$");
```

> До версії 1.9 для цієї мети слід було використовувати ім'я події зі знаком оклику `click!`, а після — ось такий workaround зав'язаний на недоліку одного регулярного виразу.

І останній випадок — видалення обробника подій прив'язаного до нашого «namespace»:

```javascript
// видаляємо всі обробники click у цьому просторі імен
$("p").off("click.namespace");
```

Хоча можна одним махом видалити всі обробники з певного простору імен:

```javascript
// обробник кліку
$("p").on("click.color", function() {
  $(this).css("color", "red");
});

// обробник при наведенні
$("p").on("mouseenter.color", function() {
  $(this).css("color", "green");
});
```

```javascript
// передумали, і все скасували
$("p").off(".color");
```

Ще корисний приклад хитрого обробника — він може ловити та обробляти дані:

```javascript
// створюємо обробник
$("p").on("click.data", function(event, one, two, three) {
    alert("Click data: " + one + ", " + two + ", " + three);
});
```

Ініціюємо обробку події, як дані передаємо масив аргументів:

```javascript
$("p").trigger("click.data", [1, 2, 3]);
```

Також хотів звернути увагу на підтримку кількох просторів імен:

```javascript
// створюємо обробник для a
$("p").on("click.a", function(event) {
    alert("Click A!");
});

// створюємо обробник для b
$("p").on("click.b", function(event) {
    alert("Click B!");
});

// створюємо обробник для простору a і b
$("p").on("click.a.b", function(event) {
    // для простору імен a і b    
    alert("Click: " + event.namespace + "!");
});
```

```javascript
// викликаємо обробник з простору a
$("p").trigger("click.a");
```

```javascript
// викликаємо обробник з простору b
$("p").trigger("click.b");
```

```javascript
// скасовуємо обробник click для простору b
$("p").off("click.b");
```
