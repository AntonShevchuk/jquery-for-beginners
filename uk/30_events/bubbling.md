# Спливання та обробка подій

Виклик більшості JavaScript-подій можна емулювати безпосередньо з самого скрипта. Ось є у нас сторінка лише з одним посиланням:

{% embed url="https://anton.shevchuk.name/book/code/events.click.html" %}

Ми можемо «клікнути» по ньому, використовуючи однойменний метод `click()`:

```javascript
$("a").get(0).click()
```

{% hint style="info" %}
Зверніть увагу на те, що я викликаю метод клік не у jQuery-об'єкта, а безпосередньо у DOM-елемента. Якщо ж ми будемо використовувати jQuery-методи `click()` або `trigger()`, то не отримаємо результату:

```javascript
$("a").click()
$("a").trigger("click")
```

Це пояснюється тим, що вони не запускають дію браузера за замовчуванням, а лише викликають обробники подій. Поверніться до цього коду, коли додасте обробники подій, він вже буде працювати.
{% endhint %}

Таким чином ми можемо прикинутися користувачем, але цього недостатньо для роботи. Нам треба навчитися відстежувати поведінку користувача та своєчасно реагувати на його дії. Почнемо з початку, візьмемо посилання і додамо до нього найпростіший обробник події `click`:

```javascript
$("a").on("click", function(event) {
    alert("Hello!")
})
```

{% hint style="info" %}
Для найбільш ходових подій існують «shorthand» методи, так, для відстеження `click` використовується `click()`, для `hover` — `hover()` і так далі :)&#x20;

У всіх наступних прикладах я буду використовувати метод `click()`
{% endhint %}

Тепер, клікнувши по посиланню, ви побачите привітання, і після закриття оного браузер перейде за посиланням, вказаним в атрибуті `href`.

_— Тільки закрив, годі вже!_

Так, це не зовсім те, що мені хотілося – треба було лише вивести текст і нікуди не йти. Ага, для цього варто скасувати дію за замовчуванням:

```javascript
$("a").click(function(event) {
    alert("Hello!");
    event.preventDefault();
})
```

Тепер переходу немає, оскільки метод `event.preventDefault()` запобігає цій дії. Але ось якщо хтось повісить ще один обробник на батьківський елемент?

```javascript
$("article").click(function(event) {
    alert("Article!");
})
```

У результаті ми отримаємо два повідомлення, але чому? Якщо у вас виникає подібне запитання, значить, ви ще не знайомі з тим, як обробляються події. Спробую коротенько дати вступну. Коли ви клікаєте на елементі в DOM-дереві, то відбувається «занурення» події – тобто спочатку всі батьківські елементи можуть обробити «клік», і лише потім він дістанеться до елемента, по якому був здійснений. Але й це ще не все. Потім подія починає проробляти зворотний шлях – «спливає», даючи тим самим другий шанс батьківським елементам обробити подію.

> Але не все так гладко — у нас же є «безсмертний» IE, який принципово не працює зі «зануренням», тому всі вирішили йти шляхом найменшого спротиву й обробляють події лише на етапі «спливання».

{% hint style="info" %}
Рекомендую до прочитання статтю «[Спливання та перехоплення](https://uk.javascript.info/bubbling-and-capturing)» з уже згаданого підручника Кантора.
{% endhint %}

Добре, наче зрозуміло, тепер повернемося до нашого прикладу, і спробуємо зрозуміти, що ж у нас відбувається. Є обробник кліку на посилання і обробник для меню з цим посиланням. При кліку на посилання спрацьовує обробник події на посиланні, і потім подія спливає до меню, і спрацьовує його обробник події «click». Але це не зовсім бажаний результат, і для боротьби з подібним шкідництвом необхідно зупиняти «спливання» подій:

```javascript
$("a").click(function(event) {
    alert("Hello!");
    event.preventDefault();
    event.stopPropagation();
})
```

Для прискорення розробки в jQuery є швидкий спосіб виклику цих двох методів за раз:

```javascript
$("a").click(function(event) {
    return false; // ось це він :)
})
```

Тепер у вас є достатній багаж знань, щоб легко маніпулювати подіями на сторінці. Хоча я додам ще трохи — для того, щоб спрацював лише ваш обробник події, можна використати метод `event.stopImmediatePropagation()`:

```javascript
$("a").click(function(event) {        // ваш обробник події
    alert("Hello!");                  // він обраний
    event.stopImmediatePropagation(); // він повинен залишитися єдиним
    return false;
})

$("a").click(function(event) { // а це чужий обробник подій
    alert("Hello again!");     // він робить все неправильно
    return false;
})
```

У цьому прикладі при кліку на посиланні буде виведено лише одне повідомлення. І так, порядок має значення.
