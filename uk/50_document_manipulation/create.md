# Створення елементів

Почнемо зі створення елементів для подальшої роботи з ними. [Документація](https://api.jquery.com/jQuery/#jQuery2) нам дбайливо повідомляє, що тут все просто:

```javascript
var $myDiv = $('<div id="my" class="some"></div>');
```

Цей приклад цілком робочий, от тільки продуктивністю він блищати не буде, адже всередині все це буде розбиратися за допомогою методу `jQuery.parseHTML()`, який зовсім не швидкий. Але ми можемо допомогти парсеру, якщо атрибути елемента будемо передавати другим параметром:

```javascript
var $myDiv = $("<div>", { "id":"my", "class":"some" });
```

Можемо зробити ще простіше:

```javascript
var $myDiv = $("<div>").attr({ "id":"my", "class":"some" });
```

І цей спосіб буде працювати на крапельку швидше. Але чому? Для того щоб відповісти на це питання, загляньте в код jQuery, у саму головну функцію `init()`. В її коді можна знайти алгоритм розбору попереднього прикладу:

1. Парсимо рядок і створюємо DOM-елемент в jQuery-обгортці
2. Заходимо в **цикл** обробки переданих параметрів:
   1. Перевіряємо, а чи немає функції у нашого елемента з такою назвою
   2. Якщо ні, то встановлюємо атрибут елемента, використовуючи метод `attr()`

Висновки робіть самі, де ми тут час утратили :)

Ну й наостанок опишу найшвидший спосіб, який я часто використовую:

```javascript
var myDiv = document.createElement("div");

myDiv.id = "my";
myDiv.className = "some";
```

Так, це «чистий» JavaScript, але, як на мене — у такому випадку він не менш зручний за будь-які фреймворки.
