# Семантична верстка

Семантична верстка HTML-документа передбачає використання тегів за прямим призначенням, тобто якщо тобі потрібен заголовок — то ось тег `<h1>` та побратими, необхідне табличне представлення даних — використовуй тег `<table>` і тільки його.

> Іноді, позбуваючись табличної верстки, доходять до абсурду, і тег `<table>` стає вигнанцем, і його перестають використовувати навіть для табличного представлення даних. Не варто повторювати цю помилку.

Забігаючи трішки наперед, варто згадати теги зі специфікації HTML5: `<article>`, `<aside>`, `<header>`, `<footer>`, `<menu>`, `<section>` тощо — використовуй їх, не бійся.

> Не боятися — це правильно, але використовувати теж треба з розумом. Рекомендую ресурс [https://html5doctor.com/](https://html5doctor.com/) — дуже добре і детально розписано про нововведення специфікації HTML5.
>
> І ще пару цікавих ресурсів на додачу:
>
> * [https://doka.guide/](https://doka.guide/) — довідник для веб-розробників
> * [https://web.dev/explore](https://web.dev/explore) — тут ціле співтовариство

Намагайся уникати надлишкових елементів на сторінці, більшість HTML-сторінок грішать зайвими блоковими елементами:

```html
<div id="header">
    <div id="logo">
        <h1><a href="/">Мій блог</a></h1>
    </div>
    <div id="description">
        <h2>Тут я ділюся своїми думками</h2>
    </div>
</div>
```

Цю конструкцію можна легко спростити, і при цьому код стане більш читабельним, зміни в CSS будуть мінімальними, або навіть не знадобляться:

```html
<header>
    <h1>
        <a href="/">Мій блог</a>
    </h1>
    <h2>Тут я ділюся своїми думками</h2>
</header>
```

> В англійській мові є термін «divits» — цим терміном нагороджують HTML-розмітку з надмірним використанням div'ів без потреби, я ж називаю такі творіння «дивними». Зазвичай цим грішать новачки, які для застосування стилів CSS обгортають елементи в div'и, що й призводить до їх розмноження без потреби.

Ще одним обов'язковим пунктом для створення «правильного» HTML є використання назв класів та ідентифікаторів, які однозначно говорять нам про вміст елемента, а не про якісь нюанси оформлення. Наведу приклад:

<table><thead><tr><th width="242">Погано</th><th></th></tr></thead><tbody><tr><td><code>red</code>, <code>green</code> тощо</td><td>в якийсь момент захочеш дизайн через CSS поміняти — і елемент з класом «red» стане синім; або возишся, бігаєш по всіх сторінках і міняєш red на blue, щоб потім ще раз почати по новій</td></tr><tr><td><code>wide</code>, <code>small</code> тощо</td><td>сьогодні широкий, а завтра?</td></tr><tr><td><code>h90w490</code></td><td>напевно, це елемент з висотою 90px і шириною 490px; чи я помиляюсь? а в смартфоні той самий клас влазить в екран?</td></tr><tr><td><code>b_1</code>, <code>px_9</code></td><td>нічого не говорять назви, які завтра вилетять з пам'яті</td></tr><tr><td><code>color1</code>, <code>color2</code>, ...</td><td>іноді зустрічається для «скінованих» сайтів, але створюються такі класи в ім'я ліні</td></tr><tr><td><code>element1</code>...<code>20</code></td><td>таке теж зустрічається і нічим добрим не пахне</td></tr></tbody></table>

І приклади правильного іменування:

<table><thead><tr><th width="245">Добре</th><th></th></tr></thead><tbody><tr><td><code>logo</code>, <code>content</code></td><td>логотип, основний контент</td></tr><tr><td><code>menu</code>, <code>submenu</code></td><td>меню та підменю</td></tr><tr><td><code>even</code>, <code>odd</code></td><td>парний та непарний елементи списку (хоча для подібних задач простіше використовувати CSS-оператори <code>:nth-child(even)</code> та <code>:nth-child(odd)</code>, знову ж таки через можливий у майбутньому зсув елементів)</td></tr><tr><td><code>navigation</code></td><td>посторінкова навігація</td></tr><tr><td><code>copyright</code></td><td>інформація про ліцензування</td></tr></tbody></table>

Є ще один момент — це форматування HTML- та CSS-коду. Я не буду загострювати на ньому увагу, але весь код в книзі буде відформатований відступами, і, можливо, це дасть свої плоди у твоїх творіннях.

> Мало не забув — не рівняйся на класи виду `p-X`, `m-X`, які ти зустрінеш в [Bootstrap](https://getbootstrap.com/) або [Tailwind](https://tailwindcss.com/). Концепція «utility first» чудово підходить для CSS фреймворків, але для розробки без використання таких подібне іменування неінформативне.
